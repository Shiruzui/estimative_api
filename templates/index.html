TaskSchema

Esquema que representa uma tarefa com nome e estimativas de tempo mínimo, provável e máximo.

  • name:

    Campo obrigatório do tipo string que representa o nome da tarefa.

  • min:

    Campo obrigatório do tipo float que representa a estimativa mínima de tempo para a tarefa.

  • likely:

    Campo obrigatório do tipo float que representa a estimativa provável de tempo para a tarefa.

  • max:

    Campo obrigatório do tipo float que representa a estimativa máxima de tempo para a tarefa.

ImageOptSchema

Esquema que representa as opções de configuração de uma imagem.

  • width:

    Campo opcional do tipo inteiro que representa a largura da imagem em pixels. O valor padrão é 16.

  • height:

    Campo pcional do tipo inteiro que representa a altura da imagem em pixels. O valor padrão é 9.

  • bins:

    Campo opcional do tipo inteiro que representa o número de barras do histograma. O valor padrão é 20.

  • density:

    Campo opcional do tipo booleano que indica se o histograma deve ser normalizado. O valor padrão é True.

  • alpha:

    Campo opcional do tipo float que representa a transparência da imagem. O valor padrão é 0.75.

  • label:

    Campo opcional do tipo string que representa o rótulo da imagem. O valor padrão é "Teste".

  • xlabel:

    Campo opcional do tipo string que representa o rótulo do eixo X da imagem. O valor padrão é "eixo X".

  • ylabel:

    Campo opcional do tipo string que representa o rótulo do eixo Y da imagem. O valor padrão é "eixo Y".

  • title:

    Campo opcional do tipo string que representa o título da imagem. O valor padrão é "Estimativa".

ImageSchema

Esquema que representa uma imagem, contendo um indicador se deve ser plotada e as opções de configuração da imagem.

  • plot:

    Campo obrigatório do tipo booleano que indica se a imagem deve ser plotada.

  • img_opt:

    Campo opcional que contém as opções de configuração da imagem. Caso não seja informado, o valor padrão é None.

  • set_default_img_opt:

    Método que é chamado após a validação dos campos e define as opções de configuração da imagem como padrão, caso não tenham sido informadas.

PostRequestSchema

Esquema que representa os dados de entrada de uma requisição POST para uma API.

  • tasks:

    Campo obrigatório que contém uma lista de tarefas no formato definido pelo TaskSchema.

  • iterations:

    Campo opcional do tipo inteiro que representa o número de iterações para cálculo das estimativas. O valor padrão é 1000.

  • type:

    Campo opcional do tipo string que representa o tipo de distribuição para cálculo das estimativas. Os valores permitidos são "triangular" e "normal". O valor padrão é "triangular".

  • percentiles:

    Campo opcional que contém uma lista de percentis a serem calculados. O valor padrão é [50, 80, 90, 100]. Cada elemento da lista deve ser um inteiro entre 0 e 100.

  • image:

    Campo obrigatório que contém as informações de plotagem da imagem no formato definido pelo ImageSchema.
